---- DESLOGUEO DE GIT EN GITHUB ---- En Windows: ir a CONTROL PANEL > ADMINISTRADOR DE CREDENCIALES > WINDOWS CREDENCIALES Buescar: git:https://github.com y eliminar. Confirmar la acción. ---- CAMBIAR USUARIO EN GIT ---- git config --global user.email"tu@email.com" git config --global user.name"tunombre" ------ ACTUALIZAR BRANCH ---------- git remote set-url origin https://github.com/your-other/repository ---- CREAR PAGE ---- Primero se sube el respositorio con el mismoNombreUsuario.github.io , luego se cambia el main por gh-pages. En "GH-PAGES" --> Ir a confirguración y deslizarse hacia abajo hasta encontrar el link: Your site is published : link ---- INICIALIZACIÓN ---- git init git add . //Si ingreso un punto, selecciona todo, sino poner nombre de carpeta git commit -m 'Nombre' git remote git remote origin main git push // Si hice alguna modificación, tomará los datos desde el pc. git pull // Si hice alguna modificación, tomará los datos desde github. ---- REINICIALIZACIÓN ---- rm -rf .git // esto eliminara todo lo relacionado a tu repositorio git git init // inicilizas git en tu proyecto git remote add origin https://gitlab.com/otro_usuario/otro_repositorio.git ----- EDITOR VIM ---- Permite modificar los comentarios que ya se hayan realizado git commit --amend :i :wq ----- TAG ---- Permite generar un antecedente de versión de programa. git tag [fecha] -m 'nombre' git log --oneline // [q para salir y h para ayuda] git push --tag ----- CLONAR ---- Baja el archivo desde el repositorio remoto. git clone urlDelProyecto git remote --v // Nos muestra el origen del respoitorio. Fetch es traer cambios nuevos. Si nosotros realizamos cambios o actualizaciones lo podremos subir al origen. git fetch // trae todos los cambios git pull origin master // trae los cambios de una rama específica. Clonar y subir proyecto a un respositorio diferente del que se clono: git init git add . git commit -m 'COMENTARIO' git push URL // En url va la url donde quiero que se haga la subida, el nuevo respositorio ----- FORK ---- Es como una clonación pero para github, es para utilizar y colaborar con otros proyectos. Se utliza para contribuir. git fork git pullRequest // avisarle al programador inicial de los cambios. git pull URL-DEL-REPO git push URL-DEL-REPO ----- RAMAS O BRANCH ---- Son varias personas que acceden a parte del mismo. Varias personas acceden a una suerte de copia con el propósito de hacer pruebas. git branch nombreElegido git branch //Me muestra la lista de ramas, el asterisco indica dónde estoy git checkout nombreElegido // Me muevo a la rama donde voy a trabajar
